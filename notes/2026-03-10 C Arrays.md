## Using an Array Name as a Pointer

These two are equivalent:
```
assume int a[] initialized

int find_largest(int a[], int n)
{
	...
}

int find_largest(int a*, int n)
{
	...
}
```
Why? --> Bc `a` is not an array, it behaves like a pointer (in the function)

if we run sizeof(a) within the above function, it returns the size of a pointer

Using separate integer pointer
```
int a[] initialized,
int *p;
p = a; (p now points to the first element in the arr)

thus,
p[1] = a[1]
p[2] = a[2]
etc
```

### Calculate sum of an array
```
#define N 10

int a[N], i, sum = 0, *p = a;

for (i = 0l i < N; i++)
	sum += p[i];
```

## Pointers and Multidimensional Arrays
**the name of the arr is the address of the first row**
- this means that `a` is no longer an integer pointer, it points to the collection of items in the first row
- equivalent is `a[0]`, the functiona; name of the multidim. array

`int a[5][5], *p=&a[0][0];`
- we must initialize p as such, because p is an integer pointer and it must point to a single integer
- `p = a` does NOT WORK (if you initialized p as an int pointer)

If we want `a[0][1]`:
- `a[0][1], *(a[0]+1), or *(p+1)`

**Int Pointers vs. Group Point**
`int (*) [10]` -> pointer to a group of 10 elements

**Examples**
- `a` -> address of `a[0]`; 1st row| Type: `int (*) [5]`
- `*a` -> value of the 1st element of 1st row
- `a[0]` -> address of first element of first row | Type: `int *`
- `&a[0]` -> same as `a` | Type: `int (*) [5]`
- `&a[0][0]` -> address of first element | Type: `int *`

`a[0] + 2` = third element of the first row
`*(a[0] + 2)` = address of the third element of the first row

## Exercises
```
int a[] = {5, 15, 34, 54, 14, 2, 52, 72};
int *p=&a[1], *q=&a[5];
```

1. `*(p+3)` = `a[1+3]=14`
2. `*(q-3)` = `a[5-3]=32`
3. `q-p` = 5-1 = 4
4. `p < q` = (1<5) = true
5. `*p<*q` = `a[1] < a[5]` = 15 < 2 = false

## A difference on `const` Position
```
void f(const int *p)
{
	int j;
	*p = 0; WRONG, VAL cannot be modified
	p = &j;
}

void fun(int* const p)
{
	int j;
	*p = 0; perfectly VALID
	p = &j; WRONG, POINTER is CONST and cannot be modified
}
```

Another example:
`i = 22`, `&i = 2020`
`int *p`, `p = 2020`

If `int * const p`:
- `p = &j` NOT Allowed because p is a constant pointer
If `const int * p`:
- `*p = 10` NOT allowed because integer value is constant

## Processing the rows
Simplification in following loop (clears row i of the array a)
```
int a[rows][cols], *p, i;
for (p = a[i]; p < a[i] + cols; p++)
	*p = 0;
```

WE can use find_largest to determine the largest element in row i of the two-dimensional array a
`largest = find_largest(a[i], cols);`

in summary, we can pass single rows by passing first address of particular row

## Processing columns
a loop that clears column i of the array a;
```
int a[rows][cols], (*p) [cols], i;
for (p = &a[0]; p < &a[rows]; p++)
	(*p) [i] = 0;
```

Tip: if p is a row pointer, p + 1 jumps 1 row,
- if p is an integer pointer, p + 1 jumps 1 integer
- etc.
