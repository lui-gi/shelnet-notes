# C Strings

## Strings in context of C

C does not have a String type, only char

C treats string literals as character arrays

Thus,
- a string literal of length `n` will be store in a `n + 1` bytes memory
- the last `char` in the array is the `null character` ('\O') as a marker for the end of the string

If we store string as an array, we can modify each char

## String Literals

`char *str="HelloWorld";`

Since string literal is stored as an array, the compiler treats it (the string literal) as a pointer of type char *
- so when `printf("abc")` is called, the address of "abc" is passed
- when `printf("%s", str)` is called, the address of "HelloWorld" is passed
